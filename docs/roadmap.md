# Roadmap — Генератор схем филейного вязания (v1)

## 1. Назначение

Этот документ описывает план реализации версии 1 генератора схем филейного вязания.

Он определяет:

* этапы разработки
* приоритеты
* порядок работы над backend и frontend
* ключевые промежуточные результаты

Roadmap оптимизирован для итеративной разработки с использованием Codex.

---

## 2. Общая стратегия

Разработка идет в следующем порядке:

1. Документация (уже выполнена)
2. Ядро backend (домен + генерация)
3. API backend
4. Минимальный frontend
5. Интеграция
6. Стабилизация

Ключевые принципы:

* Сначала backend, он задает контракт
* Frontend строго следует API backend
* Работа разбивается на маленькие проверяемые шаги

---

## 3. Этап 0 — Документация (завершён)

Артефакты:

* `product_scope.md`
* `glossary.md`
* `domain_model.md`
* `generation_rules.md`
* `api_contract_draft.md`

Результат:

* полностью определено поведение системы
* есть стабильная база для реализации

---

## 4. Этап 1 — Каркас backend

Цель:

* создать структуру проекта и базовый API

Задачи:

* инициализировать FastAPI-проект
* создать структуру:

  * `app/api`
  * `app/domain`
  * `app/services`
  * `app/core`
* реализовать `GET /health`
* описать Pydantic-модели request/response
* добавить базовую валидацию (только структура)

Результат:

* backend запускается
* есть модели данных
* логика генерации ещё не реализована

---

## 5. Этап 2 — Система глифов

Цель:

* реализовать хранение и получение глифов

Задачи:

* определить формат хранения глифов (JSON)
* создать начальный набор:

  * латиница
  * кириллица
  * цифры
  * пробел
* реализовать `GlyphRepository`
* реализовать проверку поддерживаемых символов
* написать тесты:

  * загрузка глифов
  * проверка неподдерживаемых символов

Результат:

* система умеет сопоставлять текст → глифы
* работает валидация символов

---

## 6. Этап 3 — Разбиение текста

Цель:

* преобразовать текст в строки

Задачи:

* реализовать нормализацию текста
* разбить текст на слова
* реализовать перенос строк:

  * без разрыва слов
* реализовать структуру layout (`list[string]`)
* написать тесты:

  * обычные случаи
  * крайние случаи (длинные слова, лишние пробелы)

Результат:

* система умеет преобразовывать текст → строки

---

## 7. Этап 4 — Сборка строк

Цель:

* преобразовать глифы в матрицы строк

Задачи:

* реализовать подгонку глифов под ограничения
* реализовать выравнивание по нижнему краю
* реализовать межбуквенный отступ
* склеить глифы по горизонтали
* получить `Line.cells`
* написать тесты:

  * разные размеры глифов
  * корректность отступов
  * корректность выравнивания

Результат:

* система умеет преобразовывать строки → матрицы строк

---

## 8. Этап 5 — Сборка схемы

Цель:

* собрать финальную матрицу схемы

Задачи:

* реализовать вертикальное объединение строк
* реализовать межстрочный отступ
* вычислить итоговые размеры
* добавить `null`:

  * слева/справа
  * сверху/снизу
* обеспечить прямоугольность матрицы
* написать тесты:

  * несколько строк
  * строки разной ширины
  * корректность `null`

Результат:

* система умеет преобразовывать строки → финальную схему

---

## 9. Этап 6 — Подбор варианта (поиск и scoring)

Цель:

* выбрать лучший вариант схемы

Задачи:

* реализовать перебор размеров символов
* генерировать варианты layout
* валидировать каждый вариант:

  * ограничения
  * overflow (≤ 40%)
* реализовать scoring:

  * текст помещается
  * максимальный размер символов
  * минимальное количество строк
* выбрать лучший вариант
* написать тесты:

  * сравнение вариантов
  * обработка overflow
  * крайние случаи

Результат:

* работает полный pipeline генерации

---

## 10. Этап 7 — Интеграция с API

Цель:

* подключить генерацию к API

Задачи:

* реализовать `/api/v1/schemes/generate`
* связать:

  * request → генерация → response
* преобразовать доменные модели в JSON
* реализовать обработку ошибок:

  * валидация
  * неподдерживаемые символы
  * ошибки генерации
* написать API-тесты

Результат:

* backend API полностью работает

---

## 11. Этап 8 — Frontend (минимальный)

Цель:

* создать простой интерфейс

Задачи:

* создать HTML-страницу с формой:

  * текст
  * ограничения схемы
  * ограничения символов
* собрать JSON-запрос
* вызвать backend API
* отрисовать матрицу:

  * `null` → без границы и фона
  * `0` → граница, без заливки
  * `1` → граница и заливка
* обработать ошибки
* добавить индикатор загрузки

Результат:

* есть рабочий UI

---

## 12. Этап 9 — Интеграция и тестирование

Цель:

* стабилизировать систему

Задачи:

* протестировать полный цикл:

  * ввод → backend → frontend
* проверить крайние случаи:

  * длинный текст
  * маленькие ограничения
  * overflow
* исправить несоответствия между:

  * API
  * frontend
* убрать отладочный код

Результат:

* система работает стабильно

---

## 13. Этап 10 — Улучшения (опционально)

Возможные улучшения:

* улучшить scoring
* улучшить правила отступов
* оптимизировать производительность
* улучшить ошибки
* отрефакторить код

---

## 14. Работа с Codex

Правила:

* работать строго по этапам
* не перепрыгивать этапы
* делать по одной задаче за раз
* после каждого шага:

  * проверить результат
  * сделать commit
* всегда опираться на:

  * `AGENTS.md`
  * `DOMAIN.md`
  * `GENERATION_RULES.md`
  * `API_CONTRACT.md`

---

## 15. Вехи

* работает система глифов

### Веха 3

* работает разбиение текста

### Веха 4

* работает генерация схем (без API)

### Веха 5

* работает API

### Веха 6

* подключен frontend

---

## 16. Критерий готовности (Definition of Done)

Система считается готовой, если:

* корректный input → корректная схема
* некорректный input → корректная ошибка
* output соответствует API-контракту
* frontend правильно отображает схему
* генерация соответствует `generation_rules.md`
