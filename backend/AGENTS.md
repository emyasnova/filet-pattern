# AGENTS.md

## Назначение проекта

Этот репозиторий содержит backend для генератора схем филейного вязания.

Backend принимает JSON с:

* текстом
* ограничениями схемы
* ограничениями символов

Backend возвращает схему в виде JSON.

Схема представляется как прямоугольная матрица:

* `null` — невидимая клетка
* `0` — пустая клетка
* `1` — заполненная клетка

* FastAPI backend

* перенос строк

* авторизация
* пользователи
* сохранение данных
* генерация изображений
* PDF
* frontend
* декоративные элементы
* вызовы AI

## Правила работы

Если задача находится в `tools/glyph_import`, приоритет имеют инструкции
`tools/glyph_import/AGENTS.md`. Документы основного backend читать только если
изменения затрагивают production backend, основной API или формат glyph dataset.

* Всегда читать:

  * `PRODUCT_REQUIREMENTS.md`
  * `DOMAIN.md`
  * `GENERATION_RULES.md`
  * `API_CONTRACT.md`
  * `ARCHITECTURE.md`
  * `TASKS.md`
* Не добавлять функциональность вне scope v1
* Не менять API без явного указания
* Не писать бизнес-логику в роуты
* Использовать явные доменные модели
* Использовать type hints
* Писать тесты для доменной логики
* Делать задачи строго по `TASKS.md`

## Структура кода

* `app/api` — HTTP слой
* `app/api/routes` — endpoints
* `app/api/schemas` — схемы
* `app/core` — конфиг и исключения
* `app/domain/models` — доменные модели
* `app/domain/services` — бизнес-логика
* `app/domain/rules` — правила
* `app/infrastructure/glyphs` — глифы
* `tests/unit` — unit-тесты
* `tests/integration` — интеграционные тесты
