# Glyph Import Tool

## Структура данных

Поддерживаются два сценария:

1. одна картинка = один символ
2. одна картинка = несколько символов через extraction

Актуальная модель хранения данных:

* `input/raw` — изображения, где одна картинка соответствует одному символу
* `input/extracted` — результат extraction, когда из одной картинки получаются несколько отдельных символов
* `input/debug` — debug-артефакты extraction
* `output/glyphs` — итоговые JSON глифов
* `output/manifests` — manifest-файлы

Совместимость текущего pipeline сохраняется:

* `output/previews` остаётся папкой preview-изображений
* `output/debug` остаётся папкой debug-артефактов текущего pipeline
* `output/json` остаётся legacy-путём текущего CLI; существующие JSON разложены в `output/glyphs`, но код pipeline не менялся

## Описание

Этот инструмент предназначен для преобразования фото и сканов бумажных схем букв для филейного вязания в JSON-формат глифов.

Инструмент:

* принимает изображение схемы буквы
* выполняет обработку изображения
* извлекает клеточную структуру
* определяет заполненные и пустые клетки
* формирует матрицу `0/1`
* сохраняет результат в JSON

Это offline-инструмент, не часть production backend.

## Основной сценарий

1. Положить изображение в папку `input/raw/`
2. Запустить инструмент
3. Получить:

* JSON глифа в `output/json/`
* preview в `output/previews/`
* debug-файлы в `output/debug/`

## Дополнительный сценарий: extraction символов

Инструмент также может использоваться как подготовительный этап для массового импорта глифов.

Сценарий:

1. Положить jpg/png с несколькими символами в `input/raw/`
2. Запустить extraction
3. Получить:
   * вырезанные символы в `input/extracted/`
   * debug-артефакты в `input/debug/`
   * манифест извлечения в `output/manifests/`

Этот сценарий является частью `glyph_import` и используется для подготовки glyph dataset.

Команда запуска extraction:

```bash
python -m src.main --extract --input input/raw/sheet1.jpg
```

Для других листов нужно заменить путь к файлу, например:

```bash
python -m src.main --extract --input input/raw/sheet2.jpg
python -m src.main --extract --input input/raw/sheet3.jpg
```

## Параметры `python -m src.main`

CLI поддерживает три режима запуска.

Обычный импорт одного символа:

```bash
python -m src.main --input input/raw/A.jpg --char A
```

Extraction из листа с несколькими символами:

```bash
python -m src.main --extract --input input/raw/sheet1.jpg
```

Пакетный импорт всех подходящих файлов из `input/extracted/`:

```bash
python -m src.main --batch
```

Общие параметры:

* `--input PATH` — путь к исходному `.jpg` или `.png`. Обязателен для обычного импорта и extraction. Нельзя использовать вместе с `--batch`.
* `--char CHAR` — символ, который соответствует входному изображению. Обязателен для обычного импорта. Нельзя использовать вместе с `--batch` или `--extract`.
* `--batch` — обработать все поддерживаемые `.jpg` и `.png` из `input/extracted/`. Имя файла без расширения используется как `char`; служебные extraction-файлы вида `sheet1_001.png` пропускаются.
* `--extract` — вырезать отдельные символы из одного изображения с несколькими символами. Результаты сохраняются в `input/extracted/`, debug-артефакты — в `input/debug/`, manifest — в `output/manifests/`.
* `--dry-run` — параметр принят CLI. В текущей реализации pipeline всё равно выполняется и сохраняет артефакты; флаг не отключает запись файлов.
* `--debug` — включает подробный вывод параметров и debug-логирование.

Параметры предобработки и импорта:

* `--threshold INT` — порог бинаризации изображения в диапазоне `0..255`. Значение по умолчанию: `128`.
* `--no-denoise` — отключить median denoise перед бинаризацией.
* `--denoise-size INT` — размер median-фильтра для denoise. Должен быть положительным нечётным числом. Значение по умолчанию: `3`.
* `--fill-threshold FLOAT` — доля чёрных пикселей, начиная с которой клетка считается заполненной. Диапазон: `0..1`. Значение по умолчанию: `0.35`.
* `--min-grid-step INT` — минимальный шаг линий сетки при детекции. Должен быть положительным. Значение по умолчанию: `4`.
* `--max-grid-step INT` — максимальный шаг линий сетки при детекции. Должен быть положительным и не меньше `--min-grid-step`. Значение по умолчанию: `24`.
* `--expected-columns INT` — ожидаемая ширина сетки в клетках. Если задано, вертикальные линии строятся равномерно по этому количеству колонок, без автоматического выбора их числа.
* `--expected-rows INT` — ожидаемая высота сетки в клетках. Если задано, горизонтальные линии строятся равномерно по этому количеству строк, без автоматического выбора их числа.
* `--preview-cell-size INT` — размер клетки preview-изображения в пикселях. Должен быть положительным. Значение по умолчанию: `16`.

Параметры extraction-режима:

* `--min-symbol-width INT` — минимальная ширина найденного символа в пикселях. Должна быть положительной. Значение по умолчанию: `8`.
* `--min-symbol-height INT` — минимальная высота найденного символа в пикселях. Должна быть положительной. Значение по умолчанию: `8`.
* `--min-symbol-area INT` — минимальная площадь компонента чёрных пикселей для найденного символа. Должна быть положительной. Значение по умолчанию: `64`.
* `--crop-padding INT` — padding вокруг вырезанного символа в пикселях. Не может быть отрицательным. Значение по умолчанию: `2`.

Ограничения сочетаний:

* `--extract` нельзя использовать вместе с `--batch`.
* `--batch` нельзя использовать вместе с `--input` или `--char`.
* `--extract` требует `--input`, но не принимает `--char`.
* Обычный импорт требует одновременно `--input` и `--char`.

## Пакетная обработка

Планируется отдельный batch-режим для массовой обработки всех поддерживаемых файлов из `input/extracted/` за один запуск.

Ожидаемое поведение:

* читает все файлы `.jpg` и `.png` из `input/extracted/`
* использует односимвольное имя файла без расширения как символ глифа
* пропускает служебные extraction-файлы вида `sheet1_001.png`
* прогоняет каждый файл через тот же pipeline, что и одиночный импорт
* сохраняет JSON в `output/json/`
* сохраняет preview-изображения в `output/previews/`
* сохраняет debug-артефакты в `output/debug/`
* в конце выводит сводку по успешным и ошибочным файлам

Планируемый CLI-вызов:

```bash
python -m src.main --batch
```

Примеры соответствия имени файла и символа:

* `A.jpg` -> `A`
* `С.jpg` -> `С`
* `7.png` -> `7`
